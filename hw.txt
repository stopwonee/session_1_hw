다음 문제의 답을 작성해주세요.

Q1. pull requests는 조직 레포에서 개인 레포로 보내야 한다. (y/n)
A. No. 
Pull Request는 오픈소스 협업 시, 개인 레포에서 조직 레포로 수정을 요청하는 행위이므로, 
조직 -> 개인 레포로 Pull Request를 보내야 한다는 서술은 잘못된 서술이다.

Q2. commit할 파일들이 예비로 저장되어 있는 공간의 이름은?
A. Staging. 
Working Directory에서 Staging으로 add한 내용을 commit해야 로컬 레포에 내용이 저장된다.

Q3. 조직 레포(upstream)에서 업데이트된 코드를를 로컬로 가져오고 싶다. 어떤 명령어를 사용해야 할까?
A. git pull upstream main 명령어를 이용한다. 

Q4. 오픈소스 협업으로 프로젝트를 진행한다고 할때, fork는 팀장만 해야한다. (y/n)
A. X. 
오히려 팀장은 fork 후 팀원들이 수정한 내용 간의 충돌을 조정하는 역할을 한다. 

Q5. 변경 사항이 있는 모든 파일을 "과제 끝"이라는 메시지와 함께 원격 개인 레포(origin)으로 올리고 싶다. 어떤 명령어를 사용해야 할까? 
(3개의 명령어를 전부 답해주세요.)
A. (1) git add . (2) git commi -m "과제 끝" (3) git push origin main


Q6. 현재 디렉터리를 Git 저장소로 초기화하는 명령어는?
A. git init

Q7. Git과 GitHub의 차이점은?
A. 깃은 개인 컴퓨터에 설치하는 프로그램이고, 깃허브는 깃을 지원하는 온라인 클라우드 서비스이다.

Q8. 다음 중 fork의 설명으로 올바른 것은?

1️⃣ 원본 repository에 직접 코드를 수정하는 것
2️⃣ 다른 사람의 repository를 내 계정으로 복사하는 것
3️⃣ repository를 삭제하는 것
4️⃣ branch를 생성하는 것

A. 2

Q9. 다음 Git 작업 순서 중 올바른 순서는?

1️⃣ commit → add → push
2️⃣ add → commit → push
3️⃣ push → add → commit
4️⃣ commit → push → add

A. 2

Q10. 다음 중 로컬 저장소의 변경사항을 원격 저장소로 업로드하는 명령어는?

1️⃣ git pull
2️⃣ git push
3️⃣ git clone
4️⃣ git fork

A. 2
